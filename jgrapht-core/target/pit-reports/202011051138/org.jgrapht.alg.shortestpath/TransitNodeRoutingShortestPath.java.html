/**
39 
 * Implementation of the shortest paths algorithm based on {@link TransitNodeRoutingPrecomputation}.
40 
 *
41 
 * <p>
42 
 * The algorithm is originally described the article: Arz, Julian &amp; Luxen, Dennis &amp; Sanders,
43 
 * Peter. (2013). Transit Node Routing Reconsidered. 7933. 10.1007/978-3-642-38527-8_7.
44 
 *
45 
 * <p>
46 
 * The shortest paths between vertices $u$ and $v$ is computed in the following way.
47 
 * First, a locality filter is used to determine if the vertices are local to each other.
48 
 * If so, a fallback shortest path algorithm is used to compute the path. Otherwise, there
49 
 * is a shortest path between the vertices which contains a transit vertex. Therefore the
50 
 * forward access vertices of $u$ and backward access vertices of $v$ are inspected to find
51 
 * a pair of such access vertices $(a_u, a_v)$ so that the value of $d(u,a_u) + d(a_u, a_v)
52 
 * + d(a_u, v)$ is minimum over all such pairs. Here $d(s,t)$ is the distance from vertex
53 
 * $s$ to vertex $t$.
54 
 *
55 
 * <p>
56 
 * The algorithm is designed to operate on sparse graphs with low average outdegree. Comparing
57 
 * to {@link ContractionHierarchyBidirectionalDijkstra} it uses significantly more time on the
58 
 * precomputation stage. Because of that it makes sense to use this algorithm on large instances
59 
 * (i.e. with more than 10.000 vertices), where it shows substantially better performance results
60 
 * than {@link ContractionHierarchyBidirectionalDijkstra}. Typically this algorithm is used as the
61 
 * backend for large scale shortest path search engines, e.g.
62 
 * <a href="https://www.openstreetmap.org">OpenStreetMap</a>.
63 
 *
64 
 * <p>
65 
 * The precomputation in this algorithm is performed in a lazy fashion. It can be performed
66 
 * by directly calling the {@code #performPrecomputation()} method. Otherwise, this method is called
67 
 * during the first call to either the {@code #getPath()} or {@code #getPathWeight()} methods.
68 
 *
69 
 * @param <V> graph vertex type
70 
 * @param <E> graph edge type
71 
 * @author Semen Chudakov
72 
 * @see TransitNodeRoutingPrecomputation
73 
 * @see BidirectionalDijkstraShortestPath
74 
 */
